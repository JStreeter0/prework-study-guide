HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • HTML attribute: defines additional info about element
  • Metadata:Date about our webpage that the browser uses to serve webpage user, typically not seen
  • SEO(search engine optimization): Process of improving the ranking of a website in search engines
  • Elements nested inside elements are child elements, holding element is referred to as parent element ex. head or body
  • Semantic Element:The name of the element describes its content and the role it plays on a page
  • Element/Tag difference, element is from open to close, tag is what appears in angle bracket
  • h1 should only appear once per page
  • title:defines title for webpage
  • style:defines css styles associated with page
  • base:defines base url
  • link:connects external resources to html doc
  • meta:defines metadata-character set, description, keywords, author, and viewport
  • script:element embeds client-scripts in html client
  • h1-h6:headers, H1 being largest
  • p:paragraph block
  • ul,ol,li:unordered list, ordered list, list item
  • img:info about images displayed
  • src:Attribute defines location of the image
  • a:stands for anchor,creates links to same or other webpages
  • href:attribute points to url for link
  • br:line break
  • section:semantic element that defines a section in a doc that contains thematic content

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element.
  • Css: Cascading style sheets- Programming language used to define how webpage content is presented to users
  • Three ways to style a webpage: Inline CSS, Internal CSS style sheet, and external CSS style sheet
  • Inline CSS: Style an element directly in HTML file useful for quick and permanent changes and less flexible than the other styles because each inline must be edited individually
  • External CSS: Most common way to style a webpage because developers in most cases keep all their CSS rules in separate files making design changes easier
  • What is separation of concerns? Separating code into different files based on use
  • Selector: Defines element or attribute that rules or declarations will apply to
  • Declarations include 2 important components: CSS property (we want to apply ex. color) and the value of the property (ex. blue)
  • Margin Property: indicates how much space we want around the outside of an element and the margin is not visible
  • Padding Property: Adds space around the content inside an element
  • *: Declares that ALL of the elements on the page will have the style properties be the same
  • DRY: Do not repeat yourself code- reduce the number of lines of code that need to be written or maintained
  • .card: Class selector, any HTML assigned to the class will gain the property assignments from CSS
  • CSS effects: You cas set multiple effects separated by a ;
  • Colors can be assigned by naming color or hexidecimal code/ hex #

Git

  • git add -A: This command is used to commit all files in the working branch
  • git commit -m "description here": The -m flag is used to associate a message with our commit. Commit messages should be short descriptive messsages of the changes implemented
  • git pull origin main: Good habit to always pull the base branch into the feature branch before pushing up any code, should say Already up to date once ran
  • git push origin "working branch looking to be pushed": This is used to push the changes
  • git checkout: switch to another branch and check it out into your working directory (git checkout -b branch name- creates a new branch and switches to it)
  • git status: Show modified files in working directory, staged for next commit (what branch we are currently on)

Javascript

  • JavaScript: The tool that takes static HTML/CSS webpages and makes the interactive
  • Variable: A named container that allows us to store data in our code, start with let
  • Control flow: The order in which a computer executes code in a script, can be interrupted by conditional statements and loops
  • Semicolon: Indicator of where a statement ends, only required to seperat statements on a single but good practice to have at the end of each statement
  • String: Is in single quote marks
  • Number: Have no quote marks
  • Boolean: True/False value, no quote marks
  • Array: Structure that allows you to store multiple values in a single reference
  • Object: Can be anything
  • Operator:A mathematical symbol that produces a result based on two values(variables)
  • Addition,subtraction,Multi, Division: Adds, subtracts, multiplys, or divides
  • Assignment: Assigns value to a variable(ex. let myVariable= Bob
  • Strict equality: Prefors test to see if two values equal(ex.let myt Variable =3; myVariable=== 4; True)
  • Not, Does not equal: Returns the logically opposite value of what it precedes, turns true to false and tests whether values are not equal(ex. let myVariable=3; myVariable!==3; False)
  • Conditionals: Code structures used to test if an expression is true or not(ex. ...else statement)
  • Functions: Way of packaging functionality that you wish to reuse by providing a set of instructions that tell the computer how to preform a task, functions do not auto run and must be given a name followed by ()
  • Events: Code structures that listen for activity in the browser, and run code in response
  • Primitive Values: Defined Immutable values
  • Boolean: True or False
  • Null: one value, null
  • Undefined: A variable that has not been assigned a value
  • Numeric: two types, number and BigInt alongside related value NaN
  • NaN: Not a number
  • String Type: Represents textual data- length of string dependent on #of elements, once a string is created it can not be modified but can have a substring, strings must be contained with ""
  • Symbol: Maybe used as key of object property some times called atoms
  • Objects: A value in memory, two types being data property and accessor property
  • Data Property: Associate a key with a value (value, writable, enumberable, configurable)
  • Accessor Property: Associate a key with one or two accessor functions to retrieve or store a value (get, set, enumberable, configurable)
  • Var: Declares variable
  • =: To assign the specific data that we want stored in our variable
  • console.log method: Easy way to test code
  • Conditional Statement if: A way of introducing decision-making into code, code will only execute IF the statement meets the condition or is true
  • Falsy: 0,-0,on, "", null, undefined, NaN, or a boolean data type with value of false
  • Truthy: All other values or boolean with value of true
  • Array: A single variable that is used to hold a group of data
  • Index: Location of a data item in an array, first item in an array has an index of 0
  • For loop: A loop that uses a predictable pattern to preform a task on all items in an array, has 3 important statements
  • First Statement: Starting point of loop, first index is always 0 (ex. var x = 0)
  • Second Statement: The condition (ex. x
  • Final Statement: Allows the array to iterate over each time (ex. x++ or x+1)
  • Math.floor and Math.random: Properties that can be used together to generate a random number
